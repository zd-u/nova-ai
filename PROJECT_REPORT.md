## 📋 项目概述

Nova 是一个完全开源的 AI 情感助手移动应用，采用现代化的技术栈构建。应用支持全球主流大模型生态，用户可自备任何 OpenAI 兼容格式的 API Key（如 DeepSeek、ChatGPT、Claude、Kimi 等），一键接入自己的大模型，获得温柔、有趣的陪伴体验。

**核心定位：** 提供一个完全独立、用户友好、高度可定制的 AI 情感陪伴平台，让全球用户能够自主选择和控制自己的 AI 模型。

## ✨ 核心功能特性

### 1. 智能对话系统

**功能描述：**
- Nova 采用先进的 LLM 驱动，提供自然、流畅的对话体验
- 支持多轮对话，能够理解上下文和用户意图
- 系统提示词精心设计，确保 Nova 具有傲娇、温柔、幽默的人格特征

**技术实现：**
- 后端通用 LLM 接口支持任何 OpenAI 兼容的大模型
- 前端通过 AsyncStorage 保存聊天历史
- 每条消息自动持久化，用户关闭应用后历史完整保留

**用户体验：**
- 实时消息显示，无需刷新
- 清晰的消息气泡设计，区分用户和 Nova 的消息
- 支持清除历史功能

### 2. 全模型生态支持

**支持的模型平台：**
- DeepSeek（deepseek-chat）
- OpenAI（gpt-4o、gpt-4-turbo）
- Anthropic Claude（claude-3-opus）

**实现方式：**
- 后端采用通用 LLM 适配器架构

- 支持自定义 API 地址、API Key、模型名称
- 用户配置自动保存到本地，聊天时自动加载

### 3. 本地聊天记忆

**功能描述：**
- 所有聊天记录保存在手机本地（AsyncStorage）
- 用户关闭应用后重新打开，历史消息完整保留

**存储方式：**

**功能描述：**
- 系统提示词包含多语言指令
- 自动检测用户输入的语言

**支持的语言：**

## 🔧 技术亮点

### 1. 通用 LLM 适配器架构

**设计目标：** 支持任何 OpenAI 兼容的大模型平台

**实现方式：**
```typescript
interface LLMConfig {
  apiUrl: string;      // 例如：https://api.deepseek.com/v1
  apiKey: string;      // 用户的 API Key
  modelName: string;   // 例如：deepseek-chat
}

// 后端通用调用
async function callLLM(config: LLMConfig, messages: Message[]) {
  const response = await fetch(`${config.apiUrl}/chat/completions`, {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${config.apiKey}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      model: config.modelName,
      messages,
      temperature: 0.8,
    }),
  });
  return response.json();
}
```

**优势：**
- 完全独立于任何特定的 LLM 提供商
- 用户可随时切换模型，无需修改代码
- 支持未来新增的 LLM 平台

### 2. AsyncStorage 本地持久化

**实现方式：**
- 每条消息发送/接收时自动保存
- 应用启动时自动加载历史消息
- 支持清除历史功能

**优势：**
- 完全离线可用（聊天历史部分）
- 用户数据完全在本地控制
- 无需云端数据库即可使用

### 3. 前后端分离架构

**前端职责：**
- UI 渲染和用户交互
- 本地数据存储（AsyncStorage）
- API 配置管理

**后端职责：**
- LLM API 调用
- 消息处理和转发
- 可选的数据库存储

**优势：**
- 前端可独立部署和更新
- 后端可灵活扩展和优化
- 用户可自建后端服务

- ✅ **前端：** API Key 仅保存在本地 AsyncStorage，不上传到服务器
